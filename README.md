# claude-dev-agents

開発を行うためのエージェントを作成したリポジトリです
                                                                               
  ---                                                                                            
  作成した構成                                                                                     
                                                                                                   
  スキル（スラッシュコマンド）                                                                     
                                                                                                   
  .claude/skills/develop.md — /develop で呼び出す                                                  
  - Step 1〜2（命令受取・質問）はメインコンテキストで処理
  - Step 3以降は各サブエージェントに委譲する手順を定義                                             
                                                                                                 
  サブエージェント（.claude/agents/）                                                              
                                                                                                   
  ┌─────────────────────┬────────────────────────────────────────┬─────────────────────────────┐   
  │      ファイル       │                  役割                  │          主な制約           │   
  ├─────────────────────┼────────────────────────────────────────┼─────────────────────────────┤ 
  │ issue-creator.md    │ 要件をIssue単位に細分化しGitHub登録    │ 1 Issue =                   │ 
  │                     │                                        │ 1〜3時間で完了できる粒度    │ 
  ├─────────────────────┼────────────────────────────────────────┼─────────────────────────────┤   
  │ test-case-writer.md │ TDDでテストを先行作成してコミット      │ 実装ファイルはコミットしな  │   
  │                     │                                        │ い                          │   
  ├─────────────────────┼────────────────────────────────────────┼─────────────────────────────┤   
  │ app-implementer.md  │ Issue順に実装、各Issue後にテスト確認   │ テストコードは変更禁止      │   
  ├─────────────────────┼────────────────────────────────────────┼─────────────────────────────┤
  │ test-runner-fixer.m │ 全通過まで失敗分析→修正ループを繰り返  │ 同エラー3回で諦めて報告     │   
  │ d                   │ す                                     │                             │ 
  └─────────────────────┴────────────────────────────────────────┴─────────────────────────────┘   
                                                                                                 
  サブエージェントを分ける設計の意図                                                               
  
  メインコンテキスト → Step1,2のみ（ユーザー対話が必要）                                           
  issue-creator     → Issue設計の専門コンテキスト                                                  
  test-case-writer  → TDD・テスト構造の専門コンテキスト                                            
  app-implementer   → 実装の専門コンテキスト（全Issue分のコードを持つ）                            
  test-runner-fixer → デバッグループ専用コンテキスト（大量のテスト出力を隔離）                     
                                                                                                   
  使い方                                                                                           
                                                                                                   
  /develop        
  「ユーザー管理機能付きのTodoアプリを作りたい」
  → 質問に答えるだけで Issue作成→TDDテスト→実装→テスト修正まで自動実行                             
                                                                                                   
  前提として gh コマンドの認証とGitHubリポジトリのリモート設定が必要です。
